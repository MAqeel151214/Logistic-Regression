# Logistic Regression (Perceptron Trick)

A pedagogical demonstration of binary classification using a custom implementation of the **Perceptron learning rule** (often referred to as the Perceptron trick) to approximate a logistic regression boundary. 

This repository explores how to build a basic classifier from scratch using `numpy`, understand stochastic gradient descent style weight updates, and visually inspect decision boundaries on synthetic datasets generated by `scikit-learn`.

## 📌 Project Overview

This project consists of a single Jupyter Notebook:
- `Logistic_Regression_(Perceptron_trick).ipynb`: The main notebook demonstrating environment setup, synthetic data generation, mathematical formulation, model training, and performance evaluation.

## 🧮 Mathematical Background

The core of this implementation relies on the **Perceptron Update Rule**. 

Given a linear equation for the decision boundary $z = w^T x + b$, the model predicts:
- $\hat{y} = 1$ if $z > 0$
- $\hat{y} = 0$ if $z \leq 0$

### The Perceptron Trick
During training, if the model makes an incorrect prediction, the weights are updated to "pull" the decision boundary in the correct direction:
- If predicting $1$ when the true label is $0$: $w \leftarrow w - \alpha \cdot x$
- If predicting $0$ when the true label is $1$: $w \leftarrow w + \alpha \cdot x$

where $\alpha$ is the learning rate.

### Standard Logistic Regression vs. The Perceptron Trick
While standard Logistic Regression applies the **Sigmoid activation function** $\sigma(z) = \frac{1}{1 + e^{-z}}$ and minimizes the **Log-Loss (Cross-Entropy Loss)**, this notebook deliberately explores the hard-threshold Perceptron logic. 

**Limitations of the Perceptron Trick:**
1. **Confidence Ignorance**: The perceptron rule updates only on misclassification, ignoring *how confident* the prediction is. Logistic Regression smoothly updates weights based on the difference between the predicted probability and the true label.
2. **Convergence**: A perceptron only converges if the data is perfectly linearly separable. In overlapping classes, the weights will oscillate.

## 🚀 Getting Started

### Prerequisites
Make sure you have Python installed (3.8+ recommended). 

### Setup Instructions
1. **Clone the repository**:
   ```bash
   git clone <repo-url>
   cd "Logistic Regression"
   ```

2. **Create a virtual environment (optional but recommended)**:
   ```bash
   python -m venv venv
   source venv/bin/activate  # On Windows use `venv\Scripts\activate`
   ```

3. **Install dependencies**:
   ```bash
   pip install -r requirements.txt
   ```

4. **Run the Notebook**:
   ```bash
   jupyter notebook "Logistic_Regression_(Perceptron_trick).ipynb"
   ```

## 📊 Future Improvements
As discussed in Section 8 of the notebook, the current implementation achieves ~72% accuracy due to specific technical limitations. Future improvements could include:
1. **Implementing True Logistic Regression**: Applying the Sigmoid function and computing the Log-Loss gradient.
2. **Full Epoch Iteration**: Ensuring the model samples every data point per epoch rather than a single random point.
3. **Feature Scaling**: Applying `StandardScaler` to ensure all features contribute equally to the gradient updates, leading to faster and more stable convergence.
