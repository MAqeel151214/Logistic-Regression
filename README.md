# Logistic Regression Explorations

A comprehensive pedagogical repository demonstrating binary classification through various approaches to Logistic Regression. This project explores building classifiers from scratch, understanding weight updates (via the Perceptron trick and Gradient Descent), and visually inspecting decision boundaries on synthetic datasets generated by `scikit-learn`.

## 📌 Project Overview

This repository contains three detailed Jupyter Notebooks, each focusing on a different aspect of Logistic Regression and linear classification:

1. **`Logistic_Regression_(Perceptron_trick).ipynb`**
   - Demonstrates the environment setup and synthetic data generation.
   - Explores a custom implementation of the **Perceptron learning rule** (often referred to as the Perceptron trick) using a discrete step function.
   - Covers mathematical formulation, model training, and performance evaluation.

2. **`Perceptron_trick_using_Sigmoid_Function.ipynb`**
   - Advances the perceptron trick by replacing the hard step function with the **continuous Sigmoid function**.
   - Compares the resulting decision boundaries and performance metrics against standard implementations.
   - Provides a smoother, probability-based approach to updating weights.

3. **`Logistic_Regression_using_Gradient_Descent.ipynb`**
   - Focuses on standard Logistic Regression using Gradient Descent optimization techniques (such as Stochastic Average Gradient - `sag`).
   - Utilizes `scikit-learn`'s `LogisticRegression` for model training and evaluation.
   - Includes detailed accuracy scoring and confusion matrix visualizations to thoroughly evaluate model performance on unseen test data.

## 🧮 Mathematical Background

### The Decision Boundary
Given a linear equation for the decision boundary $z = w^T x + b$, the models predict class labels based on the value of $z$.

### 1. The Perceptron Trick (Step Function)
The model predicts:
- $\hat{y} = 1$ if $z > 0$
- $\hat{y} = 0$ if $z \leq 0$

During training, if the model makes an incorrect prediction, the weights are updated to "pull" the decision boundary in the correct direction:
- True label $0$, Predicted $1$: $w \leftarrow w - \alpha \cdot x$
- True label $1$, Predicted $0$: $w \leftarrow w + \alpha \cdot x$
*(where $\alpha$ is the learning rate)*

### 2. The Sigmoid Function
Standard Logistic Regression applies the **Sigmoid activation function**:
$$ \sigma(z) = \frac{1}{1 + e^{-z}} $$
This outputs a probability between 0 and 1, allowing the model to minimize the **Log-Loss (Cross-Entropy Loss)** rather than just applying a hard threshold. This addresses the "Confidence Ignorance" limitation of the basic perceptron trick.

### 3. Gradient Descent
To find the optimal weights, the model minimizes the loss function using Gradient Descent. The weights are updated iteratively by moving in the direction of the steepest descent (negative gradient) of the loss function.

## 🚀 Getting Started

### Prerequisites
Make sure you have Python installed (Python 3.8+ is recommended).

### Setup Instructions

1. **Clone the repository**:
   ```bash
   git clone <repo-url>
   cd "Logistic Regression"
   ```

2. **Create a virtual environment (optional but highly recommended)**:
   ```bash
   python -m venv venv
   source venv/bin/activate  # On Windows use `venv\Scripts\activate`
   ```

3. **Install dependencies**:
   This project relies on several core data science libraries (listed in `requirements.txt`):
   - `numpy`
   - `pandas`
   - `scikit-learn`
   - `matplotlib`
   - `jupyter`
   
   Install them by running:
   ```bash
   pip install -r requirements.txt
   ```

4. **Launch the Notebooks**:
   ```bash
   jupyter notebook
   ```
   *Then open any of the `.ipynb` files from the Jupyter interface.*
